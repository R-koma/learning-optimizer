# 【PRD】: Learning Optimizer

## 1. Problem Statement（解決する課題）

学習（インプット）しても、それを日常的に「自分の言葉で説明する（アウトプットする）」機会が圧倒的に不足しているため、記憶が短期記憶から揮発してしまい、必要な場面で知識を引き出せないこと。

## 2. Target User（ターゲットユーザー）

- 最速で結果を出すことを目指す学習者（まずは自分自身）
- 認知負荷(短期記憶/ワーキングメモリにかかる負担)を下げ、思考力を高めたい者

## 3. Core Value（コアバリュー：何が楽になるか）

「プロテジェ効果（人に教えることで学ぶ）」をLLMとの対話によってシステム化する。
ユーザーが「学んだこと」をシステムに入力すると、深掘りLLMが的確な質問を投げかける。ユーザーはそれに答えるために「自分の頭で思い出して説明する」だけで、自ずと「バラバラだった知識が意味のある塊として整理され、最小のインプットで最大の長期記憶定着を得ることができる。

## 4. MVP Scope（必須機能：1画面 + 1フロー）

1. **インプット機能**: ユーザーが今日学んだトピック（例：「幅優先探索を学習した」）をテキスト/音声で入力する画面。
2. **学習対話フロー**: 「深掘りLLMエージェント」がユーザーの入力に対し、深掘りする質問をテキスト/音声チャット形式で返答する。
3. **ノート自動生成**: チャット/音声セッション終了時、対話内容の要約（重要な概念など）を抽出した「ノート」と「フィードバック」が自動生成され、データベースに保存される。
4. **復習**: 学習した内容を特定の時間(エビングハウスの忘却曲線)に基づき、適切なタイミングで復習をする。

## 5. Out of Scope（MVPでは「絶対にやらない」こと）

MVPの検証スピードを維持するため、以下はv1.1.0以降の要件とし、今回は実装しない。

- 外部Web検索（RAG）を用いた高度なファクトチェック機能
- SM-2アルゴリズム等の本格的な忘却曲線スケジューリング
- モバイルアプリ対応（集中環境を前提とするためPC Webのみとする）

## 6. Success Metrics（成功指標）

- **定量的指標**: エビングハウスの曲線に基づきスケジュールされた「今日の復習タスク（1日後、3日後など）」のうち、80%以上をその日のうちに実行し、対話フローを完了（ノート更新）できること。
- **定性的指標**: 学習/復習した内容を、何も見ずに構造化して説明できる実感が得られること。知識/スキルを長期記憶に保有し、必要な時に使えること。
